Project Map

How the Rainbow Tour Guides v2 codebase fits together

A quick, visual guide to the key folders, how data flows, and where to look when you want to tweak the marketplace, marketing site, or backend integrations.

Frontend runtime

Next.js 16 · App Router React 19 Tailwind 3
  • `app/` holds routes (marketing, traveler, guide, admin).
  • Layout + providers live in `app/layout.tsx` and `app/providers.tsx`.
  • UI primitives and shells are under `components/`.

Data + services

Supabase (auth + Postgres) Resend (email) Stripe (payments) Cloudinary (media)
  • `lib/data-service.ts` funnels reads/writes through Supabase.
  • Schema and triggers are defined in `supabase-schema.sql`.
  • Server/client Supabase helpers in `lib/supabase-*`.

Local dev basics

`npm install` `npm run dev` `.env.local` from `.env.example`
  • Missing Supabase creds? UI falls back to mock data in `lib/mock-data.ts`.
  • Tailwind + shadcn-style components live under `components/ui`.
  • Static assets (logos, flags, guides) live in `public/`.

How data moves through the app

Server-side work Client/UI layers
Routes

Next.js server components in `app/` fetch page data (e.g., `app/page.tsx` pulls cities/guides).

Components

Pages compose presentational + interactive pieces from `components/` (cards, filters, hero blocks, booking forms).

Data layer

`lib/data-service.ts` acts as the single entry point for queries/mutations, logging each call via `lib/query-logger.ts`.

Supabase

`lib/supabase-server.ts`/`supabase-browser.ts` create clients that talk to tables and RPCs defined in `supabase-schema.sql`.

Integrations

Emails go through `lib/email.ts` (Resend), media through `lib/cloudinary-*`, and payments through `lib/stripe.ts`.

Key app surface areas

  • `app/(marketing)/page.tsx` — public landing page.
  • `app/traveler` — traveler dashboards and trip tools.
  • `app/become-a-guide` — onboarding for guides.
  • `app/admin` — admin views and controls.
  • `app/messages` & `app/booking` — messaging and booking flows.

Supporting libraries

  • `lib/adapters.ts` — shapes Supabase rows into UI-friendly objects.
  • `lib/validations/` — zod schemas for forms.
  • `lib/hooks/` — React hooks for client interactions.
  • `scripts/` — maintenance utilities (e.g., data seeds, checks).
  • `types/` — shared TypeScript types for domain entities.

Environment + safety checklist

  • Copy `.env.example` to `.env.local` and add Supabase URL/anon key, Resend key, Stripe keys, and Cloudinary creds.
  • Secrets stay local; git ignores `.env.local` and other environment-specific files.
  • If Supabase env vars are absent, pages still render using the mock fixtures in `lib/mock-data.ts`.
Built for quick onboarding: start in `app/` to adjust pages, in `components/` to tweak UI, and in `lib/` + `supabase-schema.sql` to evolve data + integrations.
